# Capstone Grading Rubric

| Area | Weight | Strong evidence |
|---|---:|---|
| Problem definition | 10% | Clear user, need, and scope |
| Python quality | 15% | Functions, modules, typing, readable design |
| API and ingestion | 10% | Reliable retrieval, validation, error handling |
| Retrieval and RAG | 20% | Relevant context, grounded answers, source references |
| Evaluation | 15% | Repeatable dataset, metrics, failure analysis |
| Testing | 10% | Meaningful unit and integration tests |
| Deployment and usability | 10% | Reproducible setup and usable interface |
| Documentation and presentation | 10% | Clear README, diagram, demo, limitations |
