# Week 13: Testing

## Objective

Use pytest, fixtures, mocks, and integration tests to make the application reliable.

## Required Output

Add tests for API, chunking, retrieval, and answer-format components.

## Four-Day Schedule

### Day 1 — Concept and Guided Practice

- **0–10 minutes:** Review the prior week and open the repository.
- **10–25 minutes:** Unit tests.
- **25–50 minutes:** Follow the guided example and run it locally.
- **50–60 minutes:** Record notes and commit the work.

### Day 2 — Coding Practice

- **0–10 minutes:** Reproduce yesterday's main idea without notes.
- **10–25 minutes:** Fixtures and parameterization.
- **25–50 minutes:** Complete the coding exercise.
- **50–60 minutes:** Run checks, fix errors, and commit.

### Day 3 — Independent Build

- **0–10 minutes:** Define the session's small deliverable.
- **10–25 minutes:** Mock external APIs.
- **25–50 minutes:** Add the idea to the weekly project.
- **50–60 minutes:** Document decisions and commit.

### Day 4 — Testing and Communication

- **0–10 minutes:** Review unfinished work.
- **10–25 minutes:** Continuous integration.
- **25–45 minutes:** Test and improve the deliverable.
- **45–55 minutes:** Write a short reflection:
    - What did I build?
    - What failed?
    - How did I fix it?

- **55–60 minutes:** Push the final weekly commit.
