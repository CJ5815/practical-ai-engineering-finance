# Interview Preparation

A certificate says that you completed material. A project demonstrates that
you can apply it.

## Be Ready to Explain

- Why you selected the problem.
- How information moves through the system.
- Why you used an API, vector database, and RAG.
- How you prevented unsupported answers.
- How you evaluated the application.
- One difficult bug and how you solved it.
- A limitation you would address next.

## Two-Minute Project Explanation

Use this structure:

1. **Problem:** What user problem did you address?
2. **Approach:** What did you build?
3. **Technology:** Why did you choose the tools?
4. **Evidence:** What did testing and evaluation show?
5. **Reflection:** What would you improve?
